## 6. 決定木 — ネストされた判定ルールの「不合理なほどの」パワー
**原題**: Decision trees – the unreasonable power of nested decision rules

**ざっくり言うと:**
Amazon（MLU: Machine Learning University）が公開している（決定木（Decision Tree）: データを「はい/いいえ」の質問の連鎖で分類していくアルゴリズム。「年齢は30歳以上？」→「収入は500万以上？」のように条件分岐を繰り返して予測する）のインタラクティブな解説ページがHNで話題になった。美しいビジュアライゼーションで（エントロピー: データの「混ざり具合」を数値で表したもの。よく混ざっている＝エントロピーが高い＝まだ分類が不十分）や分割の仕組みを直感的に理解できる。

**もう少し詳しく:**
決定木はデータを上から下に向かって順番に分割していく。各ステップで「どの特徴量を使って、どこで分割すれば最も効果的か？」を決めるのにエントロピーを使う。このページではアニメーションと対話的なUI で、分割の各ステップがデータにどう影響するかを可視化している。決定木の強みは「なぜその判定になったか」が追跡できる（説明可能性: AIの判断理由を人間が理解できること。ブラックボックスなニューラルネットワークと対照的な性質）こと。CERNの物理実験では2010年頃にニューラルネットワークより決定木ベースの分類器（Boosted Decision Trees）が好まれていたのも、この説明可能性が理由だった。面白い指摘として「1ビットのニューラルネットワークは決定木である」というコメントがあり、理論上はほとんどのニューラルネットワークをif-else文の連鎖に「コンパイル」できるが、いつそのアプローチがうまくいくかはまだよくわかっていないという。

**なぜ注目？:**
LLMやディープラーニングが注目を集める中、「古典的だが強力な」機械学習手法を再評価する動き。決定木は推論速度が圧倒的に速く、あるコメントでは「小さなニューラルネットワークで置き換えようとしたが、分類精度は上がっても推論速度は2桁遅くなった」と報告されている。低レイテンシが求められるアプリケーションでは、今でも決定木が最適解になりうる。

## 用語集

- **GPT（Generative Pre-trained Transformer）**: テキストを生成するAIモデルの設計思想。ChatGPTやClaudeの基盤技術
- **MCP（Model Context Protocol）**: AIエージェントが外部ツールと連携するための標準規格。Anthropicが提唱した
- **CLI（Command Line Interface）**: コマンドを打ち込んでコンピュータを操作する仕組み。ターミナルで使うツール全般
- **決定木（Decision Tree）**: データを「はい/いいえ」の質問の連鎖で分類する機械学習アルゴリズム。説明可能性と推論速度が強み
- **ナノ材料**: ナノメートルサイズの極めて小さな材料。通常の材料とは異なる特殊な性質を持ち、医療分野で注目されている
- **ヘッドレス**: GUI（画面）を持たず、コマンドラインやAPIだけで動作するソフトウェアの形態
- **XML（eXtensible Markup Language）**: データを構造化するためのマークアップ言語。`<tag>内容</tag>` の形式で情報を整理する
- **プロンプトエンジニアリング**: AIから望ましい出力を得るための入力（プロンプト）の設計技術
- **LLM（Large Language Model）**: 大量のテキストデータで学習した大規模AIモデル。ChatGPT、Claude、Geminiなどの総称
- **エントロピー**: 情報理論における「不確実さ」の度合い。機械学習ではデータの混ざり具合を測る指標として使われる
